ТЗ: Кнопки управления игрой (Telegram Keyboard)

1) Цель

Упростить управление ботом: основные действия доступны кнопками внизу чата, без ввода команд. Пользователь должен уметь:
	•	начать/перезапустить игру,
	•	получить инструкцию,
	•	запросить новое слово (новая игра),
	•	сдаться,
	•	посмотреть статус/попытки (опционально).

⸻

2) UX и список кнопок

2.1 Основные кнопки (обязательно)

Кнопки (текст на кнопке):
	1.	🎮 Новая игра — начать новую игру с новым словом
	2.	🔄 Рестарт — начать новую игру (синоним “Новая игра”; можно оставить одну кнопку, но в ТЗ обе для явности)
	3.	ℹ️ Инструкция — показать правила + команды
	4.	🏳️ Сдаться — завершить текущую игру и показать ответ

2.2 Дополнительные (опционально)
	5.	📊 Статус — показать: номер попытки, историю последних попыток (или сколько осталось)
	6.	📈 Статистика — если есть учёт побед/поражений

2.3 Раскладка клавиатуры (рекомендуется)

ReplyKeyboard (не inline), 2 ряда:
	•	Ряд 1: 🎮 Новая игра | 🔄 Рестарт
	•	Ряд 2: ℹ️ Инструкция | 🏳️ Сдаться
	•	(если есть) Ряд 3: 📊 Статус | 📈 Статистика

Параметры клавиатуры:
	•	resize_keyboard = True
	•	one_time_keyboard = False (клавиатура всегда видна)
	•	input_field_placeholder = "Введите слово из 5 букв" (если поддерживается библиотекой)

⸻

3) Логика поведения кнопок

3.1 Общие правила
	•	Нажатие кнопки отправляет в бот обычное текстовое сообщение (например “🎮 Новая игра”), поэтому бот должен обрабатывать эти тексты как команды.
	•	Кнопки должны быть доступны всегда (в любой стадии игры).

3.2 “ℹ️ Инструкция”
	•	Бот отвечает инструкцией (правила + обозначения 🟩🟨⬛ + список действий).
	•	Игра не сбрасывается.

Критерий приёмки: кнопка всегда выводит правила, независимо от активной игры.

3.3 “🎮 Новая игра” / “🔄 Рестарт”
	•	Бот сбрасывает текущую игру (если была) и создаёт новую с новым загаданным словом.
	•	Бот пишет: “Новая игра началась! Введите слово из 5 букв.”
	•	(Опционально) если была активная игра — можно добавить “Предыдущая игра завершена.”

Критерий приёмки: после нажатия ввод слова сравнивается уже с новым словом.

3.4 “🏳️ Сдаться”
	•	Если игры нет: “Сейчас нет активной игры. Нажмите ‘Новая игра’.”
	•	Если игра есть: бот выводит “Поражение/Игра завершена. Слово дня: XXXXX” и помечает игру завершённой.
	•	После сдачи бот предлагает начать заново (сообщением) и клавиатура остаётся.

Критерий приёмки: после “Сдаться” следующая попытка не принимается как продолжение старой игры (нужно начать новую).

3.5 “📊 Статус” (опционально)
	•	Показывает:
	•	текущую попытку (например “Попытка 3 из 6”)
	•	историю попыток (слово + квадраты) или хотя бы последние 1–3
	•	сколько попыток осталось

⸻

4) Интеграция: когда показывать клавиатуру

4.1 Показ клавиатуры

Клавиатура должна прикрепляться к ответам бота в случаях:
	•	после /start
	•	после /help
	•	после начала новой игры
	•	после сдачи/победы/поражения
	•	после ответа на попытку (результат с квадратами)

Критерий приёмки: после любого “основного” сообщения бота клавиатура остаётся видимой.

4.2 Совместимость с командами
	•	Текстовые команды /start, /help, /restart, /giveup должны продолжать работать.
	•	Кнопки — альтернативный UX.

Критерий приёмки: и команды, и кнопки дают одинаковый результат.

⸻

5) Конфигурация

5.1 Переменная окружения

Добавить настройку:
	•	USE_KEYBOARD=1 (по умолчанию 1)

Если USE_KEYBOARD=0, бот работает как сейчас без клавиатуры (для отладки/минимализма).

6) Тестирование

6.1 Unit-тесты
	•	функция/маппинг “текст кнопки → действие”
	•	старт новой игры по кнопке
	•	сдача по кнопке
	•	инструкция по кнопке

6.2 Критерии приёмки (ручные)
	1.	Нажимаю ℹ️ Инструкция → вижу правила, клавиатура на месте.
	2.	Нажимаю 🎮 Новая игра → игра стартует, ввод слова даёт квадраты.
	3.	Нажимаю 🏳️ Сдаться → бот показывает ответ, игра завершается.
	4.	После победы/поражения клавиатура остаётся, можно сразу нажать 🎮 Новая игра.
