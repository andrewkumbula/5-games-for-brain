# ТЗ: Визуализация истории попыток (Telegram Wordle «5 букв»)

Цель этого документа — зафиксировать **единый UX-паттерн визуализации игры** Wordle в Telegram:  
вся история попыток отображается **в одном сообщении**, а **каждая буква подсвечена напрямую**, чтобы пользователь сразу понимал результат без сопоставлений.

Документ предназначен для прямой передачи в реализацию.

---

## 1. Цель и мотивация

Текущая визуализация вида:

```
СЛОВО
⬛🟩🟨⬛🟩
```

создаёт высокую когнитивную нагрузку:
- пользователю нужно сопоставлять буквы и квадраты,
- сложно анализировать несколько попыток,
- игра плохо читается на мобильных устройствах.

Цель нового формата:
- убрать необходимость «думать глазами»,
- сделать каждую попытку самодостаточной,
- хранить всю историю игры в одном месте.

---

## 2. Основной формат (ОБЯЗАТЕЛЬНЫЙ)

### 2.1 Формат одной попытки

Каждая попытка отображается **одной строкой**, где **каждая буква сразу окрашена**:

```
⬛П🟩Р🟩У⬛Н🟩А
```

Правила:
- 🟩 — буква на правильном месте
- 🟨 — буква есть в слове, но в другой позиции
- ⬛ — буквы нет в слове
- буквы всегда **заглавные**
- цветной квадрат идёт **непосредственно перед буквой**
- пробелы не используются

---

## 3. История попыток (одно сообщение)

### 3.1 Общий формат сообщения

Во время игры бот поддерживает **одно сообщение**, которое редактируется после каждой попытки.

Пример:

```
🎯 Игра «5 букв»

⬛П🟩Р🟩У⬛Н🟩А
⬛С⬛Т🟩У⬛Ж🟩А
⬛К🟩Р🟩У⬛П🟩А

Попытка: 3 / 6
```

### 3.2 Обязательные элементы
Сообщение должно содержать:
1. Заголовок игры
2. Историю всех попыток (сверху вниз)
3. Текущий статус: номер попытки и лимит

---

## 4. Поведение бота

### 4.1 Принцип «одно сообщение — одна игра»
- бот **не отправляет новое сообщение** после каждой попытки
- бот **редактирует существующее сообщение** (`editMessageText`)
- новая попытка → пересборка всего текста → редактирование

Это:
- не засоряет чат,
- позволяет сразу видеть прогресс,
- улучшает восприятие игры.

---

## 5. Победа и поражение

### 5.1 Победа

Если слово угадано:

```
🎉 Победа!

🟩Г🟩Р🟩У🟩Ш🟩А

Попыток использовано: 4 / 6
```

Требования:
- история попыток сохраняется
- сообщение становится финальным
- отображаются кнопки: «🎮 Новая игра»

---

### 5.2 Поражение

Если попытки закончились:

```
💥 Поражение

⬛П🟩Р🟩У⬛Н🟩А
⬛С⬛Т🟩У⬛Ж🟩А
⬛К🟩Р🟩У⬛П🟩А
⬛Л🟨У⬛Н🟩А
⬛К⬛Р⬛У⬛П⬛А
⬛П⬛Р⬛У⬛Н⬛А

Слово дня: ГРУША
```

Требования:
- история полностью сохранена
- слово-ответ показано явно
- доступна кнопка «🎮 Новая игра»

---

## 6. Кнопки управления

Кнопки располагаются **под сообщением истории** и доступны всегда:

- 🎮 Новая игра
- ℹ️ Инструкция
- 🏳️ Сдаться

Поведение:
- кнопки не сбрасывают историю сами по себе
- «Новая игра» создаёт новое сообщение
- «Сдаться» завершает текущую игру и редактирует сообщение

---

## 7. Алгоритм формирования строки

Для каждой попытки:

Входные данные:
- `guess`: строка длиной 5 (в нижнем регистре)
- `states`: массив состояний `[ABSENT, PRESENT, CORRECT, ...]`

Выход:
```
⬛П🟩Р🟩У⬛Н🟩А
```

Правило:
- для каждого индекса `i`:
  - взять цвет по `states[i]`
  - взять `guess[i].upper()`
  - склеить в одну строку

---

## 8. Причины выбора решения

Данный формат:
- снижает когнитивную нагрузку
- делает игру визуально «читаемой»
- не требует сложной графики или canvas
- стабильно работает в Telegram
- легко масштабируется (статистика, история, алфавит)

Это рекомендуемый **best practice** для Wordle-подобных ботов в мессенджерах.

---

## 9. Критерии приёмки

Реализация считается принятой, если:
- каждая попытка отображается одной строкой с подсвеченными буквами
- история хранится и обновляется в одном сообщении
- пользователь без объяснений понимает, какие буквы верны
- чат не захламляется множеством сообщений
