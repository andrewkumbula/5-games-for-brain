# ТЗ: CLI для сборки и обновления словаря (Telegram Wordle «5 букв»)

Цель этого документа — описать **в одном файле** все требования к CLI-утилите, которая собирает, нормализует и сохраняет словарь русских слов из 5 букв для Telegram-бота Wordle.

---

## Цель

CLI-утилита должна автоматически:
- загружать словари слов,
- отфильтровывать и нормализовать слова,
- сохранять их в `words.json` и/или в SQLite,
- обеспечивать корректную работу игры без ошибок «слова нет в словаре».

Утилита используется разработчиком или CI, а не конечным пользователем бота.

---

## Формат запуска

CLI должен запускаться одной командой из корня проекта, например:

```bash
python -m tools.dict build
```

Допустим альтернативный запуск:

```bash
python tools/dict_cli.py build
```

---

## Команды CLI

CLI поддерживает следующие команды:

### build

Основная команда сборки словаря.

Пример:
```bash
python -m tools.dict build --out json
```

Параметры:
- `--out json|db|both` — куда сохранять словарь (по умолчанию `json`)
- `--words-path PATH` — путь к `words.json`
- `--db-path PATH` — путь к SQLite базе
- `--source-url URL` — основной источник слов
- `--extra-source-url URL` — дополнительный источник (опционально)
- `--yo to_e|keep` — нормализация `ё` (по умолчанию `to_e`)
- `--answers-size N` — размер списка слов-ответов
- `--seed INT` — seed для детерминированной выборки
- `--min-allowed INT` — минимально допустимый размер словаря (по умолчанию 10000)
- `--dry-run` — не сохранять результат, только показать статистику

CLI обязан вывести:
- источники слов
- общее число строк
- число слов после фильтрации
- количество `allowed`
- количество `answers`
- путь сохранения

Если `allowed < min-allowed`, команда должна завершаться ошибкой.

---

### validate

Проверяет существующий словарь.

Команда:
```bash
python -m tools.dict validate
```

Проверки:
- корректность структуры JSON / БД
- `answers ⊆ allowed`
- все слова состоят из 5 кириллических букв
- словарь не пустой и превышает минимальный размер

---

### stats (опционально)

Показывает статистику словаря:
- количество слов
- первые и последние N слов
- наличие символа `ё`
- дату последнего обновления (если есть)

---

## Источники слов

Основной источник:
- готовый Wordle-словарь русских слов из 5 букв (URL задаётся в коде или флагом)

Дополнительные источники могут подключаться через `--extra-source-url`.

Все источники проходят одинаковую фильтрацию и дедупликацию.

---

## Нормализация и фильтрация

Каждое слово должно быть обработано следующим образом:
1. Удаление пробелов по краям
2. Приведение к нижнему регистру
3. Нормализация `ё` → `е` (если включено)
4. Проверка: только символы `[а-я]`
5. Проверка: длина ровно 5 символов
6. Удаление дубликатов
7. Сортировка перед сохранением

Слова, не прошедшие проверки, отбрасываются.

---

## Форматы хранения

### JSON

Файл по умолчанию: `words.json`

Формат:
```json
{
  "answers": ["улица", "автор"],
  "allowed": ["автор", "улица"]
}
```

Требования:
- `answers` всегда подмножество `allowed`
- по умолчанию `answers = allowed`
- сортировка обязательна

---

### SQLite

Таблицы:
- `words_allowed(word TEXT PRIMARY KEY)`
- `words_answers(word TEXT PRIMARY KEY)`
- `meta(key TEXT PRIMARY KEY, value TEXT)`

Режимы:
- `replace` — очистить и перезаписать (по умолчанию)
- `upsert` — добавить новые слова

---

## Интеграция с ботом

- Бот берёт слово дня из `answers`
- Проверка ввода идёт по `allowed`
- При отсутствии словаря бот логирует ошибку и сообщает, что нужно запустить CLI
- В мягком режиме бот принимает любое валидное слово из 5 букв

---

## Тестирование

Unit-тесты должны покрывать:
- нормализацию (`ё`, регистр)
- фильтрацию по длине и алфавиту
- дедупликацию
- корректность `answers ⊆ allowed`
- детерминированность `seed`
- `dry-run`

Интеграционный тест:
- сборка словаря
- валидация
- проверка минимального размера

## Критерии приёмки

- Словарь собирается одной командой
- В `allowed` десятки тысяч слов
- Бот принимает распространённые слова
- JSON стабилен и предсказуем в git
- `validate` проходит без ошибок
